29Matplotlib date format 

30---------------------- 

31 

32Matplotlib represents dates using floating point numbers specifying the number 

33of days since a default epoch of 1970-01-01 UTC; for example, 

341970-01-01, 06:00 is the floating point number 0.25. The formatters and 

35locators require the use of `datetime.datetime` objects, so only dates between 

36year 0001 and 9999 can be represented. Microsecond precision 

37is achievable for (approximately) 70 years on either side of the epoch, and 

3820 microseconds for the rest of the allowable range of dates (year 0001 to 

399999). The epoch can be changed at import time via `.dates.set_epoch` or 

40:rc:`dates.epoch` to other dates if necessary; see 

41:doc:`/gallery/ticks/date_precision_and_epochs` for a discussion. 

42 

43.. note:: 

44 

45 Before Matplotlib 3.3, the epoch was 0000-12-31 which lost modern 

46 microsecond precision and also made the default axis limit of 0 an invalid 

47 datetime. In 3.3 the epoch was changed as above. To convert old 

48 ordinal floats to the new epoch, users can do:: 

49 

50 new_ordinal = old_ordinal + mdates.date2num(np.datetime64('0000-12-31'))

69.. note:: 

70 

71 Like Python's `datetime.datetime`, Matplotlib uses the Gregorian calendar 

72 for all conversions between dates and floating point numbers. This practice 

73 is not universal, and calendar differences can cause confusing 

74 differences between what Python and Matplotlib give as the number of days 

75 since 0001-01-01 and what other software and databases yield. For 

76 example, the US Naval Observatory uses a calendar that switches 

77 from Julian to Gregorian in October, 1582. Hence, using their 

78 calculator, the number of days between 0001-01-01 and 2006-04-01 is 

79 732403, whereas using the Gregorian calendar via the datetime 

80 module we find:: 

81 

82 In [1]: date(2006, 4, 1).toordinal() - date(1, 1, 1).toordinal() 

83 Out[1]: 732401 

84 

85All the Matplotlib date converters, tickers and formatters are timezone aware. 

86If no explicit timezone is provided, :rc:`timezone` is assumed, provided as a 

87string. If you want to use a different timezone, pass the *tz* keyword 

88argument of `num2date` to any date tickers or locators you create. This can 

89be either a `datetime.tzinfo` instance or a string with the timezone name that 

90can be parsed by `~dateutil.tz.gettz`. 

91 

92A wide range of specific and general purpose date tick locators and 

93formatters are provided in this module. See 

94:mod:`matplotlib.ticker` for general information on tick locators 

95and formatters. These are described below. 

96 

97The dateutil_ module provides additional code to handle date ticking, making it 

98easy to place ticks on any kinds of dates. See examples below. 

99 

100.. _dateutil: https://dateutil.readthedocs.io

102Date tickers 

103------------ 

104 

105Most of the date tickers can locate single or multiple values. For example:: 

106 

107 # import constants for the days of the week 

108 from matplotlib.dates import MO, TU, WE, TH, FR, SA, SU 

109 

110 # tick on Mondays every week 

111 loc = WeekdayLocator(byweekday=MO, tz=tz) 

112 

113 # tick on Mondays and Saturdays 

114 loc = WeekdayLocator(byweekday=(MO, SA)) 

115 

116In addition, most of the constructors take an interval argument:: 

117 

118 # tick on Mondays every second week 

119 loc = WeekdayLocator(byweekday=MO, interval=2) 

120 

121The rrule locator allows completely general date ticking:: 

122 

123 # tick every 5th easter 

124 rule = rrulewrapper(YEARLY, byeaster=1, interval=5) 

125 loc = RRuleLocator(rule) 

126 

127The available date tickers are: 

128 

129* `MicrosecondLocator`: Locate microseconds. 

130 

131* `SecondLocator`: Locate seconds. 

132 

133* `MinuteLocator`: Locate minutes. 

134 

135* `HourLocator`: Locate hours. 

136 

137* `DayLocator`: Locate specified days of the month. 

138 

139* `WeekdayLocator`: Locate days of the week, e.g., MO, TU. 

140 

141* `MonthLocator`: Locate months, e.g., 7 for July. 

142 

143* `YearLocator`: Locate years that are multiples of base. 

144 

145* `RRuleLocator`: Locate using a `rrulewrapper`. 

146 `rrulewrapper` is a simple wrapper around dateutil_'s `dateutil.rrule` 

147 which allow almost arbitrary date tick specifications. 

148 See :doc:`rrule example </gallery/ticks/date_demo_rrule>`. 

149 

150* `AutoDateLocator`: On autoscale, this class picks the best `DateLocator` 

151 (e.g., `RRuleLocator`) to set the view limits and the tick locations. If 

152 called with ``interval_multiples=True`` it will make ticks line up with 

153 sensible multiples of the tick intervals. For example, if the interval is 

154 4 hours, it will pick hours 0, 4, 8, etc. as ticks. This behaviour is not 

155 guaranteed by default. 

156 

157Date formatters 

158--------------- 

159 

160The available date formatters are: 

161 

162* `AutoDateFormatter`: attempts to figure out the best format to use. This is 

163 most useful when used with the `AutoDateLocator`. 

164 

165* `ConciseDateFormatter`: also attempts to figure out the best format to use, 

166 and to make the format as compact as possible while still having complete 

167 date information. This is most useful when used with the `AutoDateLocator`. 

168 

169* `DateFormatter`: use `~datetime.datetime.strftime` format strings.

321def _dt64_to_ordinalf(d): 

322 """ 

323 Convert `numpy.datetime64` or an `numpy.ndarray` of those types to 

324 Gregorian date as UTC float relative to the epoch (see `.get_epoch`). 

325 Roundoff is float64 precision. Practically: microseconds for dates 

326 between 290301 BC, 294241 AD, milliseconds for larger dates 

327 (see `numpy.datetime64`). 

328 """ 

329 

330 # the "extra" ensures that we at least allow the dynamic range out to 

331 # seconds. That should get out to +/-2e11 years. 

332 dseconds = d.astype('datetime64[s]') 

333 extra = (d - dseconds).astype('timedelta64[ns]') 

334 t0 = np.datetime64(get_epoch(), 's') 

335 dt = (dseconds - t0).astype(np.float64) 

336 dt += extra.astype(np.float64) / 1.0e9 

337 dt = dt / SEC_PER_DAY 

338 

339 NaT_int = np.datetime64('NaT').astype(np.int64) 

340 d_int = d.astype(np.int64) 

341 dt[d_int == NaT_int] = np.nan 

342 return dt

345def _from_ordinalf(x, tz=None): 

346 """ 

347 Convert Gregorian float of the date, preserving hours, minutes, 

348 seconds and microseconds. Return value is a `.datetime`. 

349 

350 The input date *x* is a float in ordinal days at UTC, and the output will 

351 be the specified `.datetime` object corresponding to that time in 

352 timezone *tz*, or if *tz* is ``None``, in the timezone specified in 

353 :rc:`timezone`. 

354 """ 

355 

356 tz = _get_tzinfo(tz) 

357 

358 dt = (np.datetime64(get_epoch()) + 

359 np.timedelta64(int(np.round(x * MUSECONDS_PER_DAY)), 'us')) 

360 if dt < np.datetime64('0001-01-01') or dt >= np.datetime64('10000-01-01'): 

361 raise ValueError(f'Date ordinal {x} converts to {dt} (using ' 

362 f'epoch {get_epoch()}), but Matplotlib dates must be ' 

363 'between year 0001 and 9999.') 

364 # convert from datetime64 to datetime: 

365 dt = dt.tolist() 

366 

367 # datetime64 is always UTC: 

368 dt = dt.replace(tzinfo=dateutil.tz.gettz('UTC')) 

369 # but maybe we are working in a different timezone so move. 

370 dt = dt.astimezone(tz) 

371 # fix round off errors 

372 if np.abs(x) > 70 * 365: 

373 # if x is big, round off to nearest twenty microseconds. 

374 # This avoids floating point roundoff error 

375 ms = round(dt.microsecond / 20) * 20 

376 if ms == 1000000: 

377 dt = dt.replace(microsecond=0) + datetime.timedelta(seconds=1) 

378 else: 

379 dt = dt.replace(microsecond=ms) 

380 

381 return dt

416def date2num(d): 

417 """ 

418 Convert datetime objects to Matplotlib dates. 

419 

420 Parameters 

421 ---------- 

422 d : `datetime.datetime` or `numpy.datetime64` or sequences of these 

423 

424 Returns 

425 ------- 

426 float or sequence of floats 

427 Number of days since the epoch. See `.get_epoch` for the 

428 epoch, which can be changed by :rc:`date.epoch` or `.set_epoch`. If 

429 the epoch is "1970-01-01T00:00:00" (default) then noon Jan 1 1970 

430 ("1970-01-01T12:00:00") returns 0.5. 

431 

432 Notes 

433 ----- 

434 The Gregorian calendar is assumed; this is not universal practice. 

435 For details see the module docstring. 

436 """ 

437 # Unpack in case of e.g. Pandas or xarray object 

438 d = cbook._unpack_to_numpy(d) 

439 

440 # make an iterable, but save state to unpack later: 

441 iterable = np.iterable(d) 

442 if not iterable: 

443 d = [d] 

444 

445 masked = np.ma.is_masked(d) 

446 mask = np.ma.getmask(d) 

447 d = np.asarray(d) 

448 

449 # convert to datetime64 arrays, if not already: 

450 if not np.issubdtype(d.dtype, np.datetime64): 

451 # datetime arrays 

452 if not d.size: 

453 # deals with an empty array... 

454 return d 

455 tzi = getattr(d[0], 'tzinfo', None) 

456 if tzi is not None: 

457 # make datetime naive: 

458 d = [dt.astimezone(UTC).replace(tzinfo=None) for dt in d] 

459 d = np.asarray(d) 

460 d = d.astype('datetime64[us]') 

461 

462 d = np.ma.masked_array(d, mask=mask) if masked else d 

463 d = _dt64_to_ordinalf(d) 

464 

465 return d if iterable else d[0]

568def drange(dstart, dend, delta): 

569 """ 

570 Return a sequence of equally spaced Matplotlib dates. 

571 

572 The dates start at *dstart* and reach up to, but not including *dend*. 

573 They are spaced by *delta*. 

574 

575 Parameters 

576 ---------- 

577 dstart, dend : `~datetime.datetime` 

578 The date limits. 

579 delta : `datetime.timedelta` 

580 Spacing of the dates. 

581 

582 Returns 

583 ------- 

584 `numpy.array` 

585 A list floats representing Matplotlib dates. 

586 

587 """ 

588 f1 = date2num(dstart) 

589 f2 = date2num(dend) 

590 step = delta.total_seconds() / SEC_PER_DAY 

591 

592 # calculate the difference between dend and dstart in times of delta 

593 num = int(np.ceil((f2 - f1) / step)) 

594 

595 # calculate end of the interval which will be generated 

596 dinterval_end = dstart + num * delta 

597 

598 # ensure, that an half open interval will be generated [dstart, dend) 

599 if dinterval_end >= dend: 599 ↛ 605line 599 didn't jump to line 605, because the condition on line 599 was never false

600 # if the endpoint is greater than or equal to dend, 

601 # just subtract one delta 

602 dinterval_end -= delta 

603 num -= 1 

604 

605 f2 = date2num(dinterval_end) # new float-endpoint 

606 return np.linspace(f1, f2, num + 1)

609def _wrap_in_tex(text):
